## Conexi√≥n a una Base De Datos

Seg√∫n los est√°ndares prescritos, el primer paso del proceso es obtener la conexi√≥n con el objeto que representa la base de datos.

### M√©todos de un objeto Conexi√≥n

El m√©todo `connect()` devuelve un objeto de conexi√≥n que hace referencia a la base de datos existente o una nueva base de datos si no existe.

Los siguientes m√©todos del objeto conexi√≥n son √∫tiles:

|M√©todo|Descripci√≥n|
|:-----|:----------|
|`cursor()`|Devuelve un objeto cursor que usa esta conexi√≥n.|
|`commit()`|Compromete expl√≠citamente cualquier transacci√≥n pendiente a la base de datos.|
|`rollback()`|Este m√©todo opcional hace que una transacci√≥n se retrotraiga al punto de partida.|
|`close()`|Cierra la conexi√≥n a la base de datos de forma permanente.|

Los m√©todos `commit()` y `rollback()` del objeto de conexi√≥n garantizan el control de transacciones. Ejemplo:

```python

  db.execute("Query")
  db.commit() # confirmamos expl√≠citamente los cambios
else:
  print("Error")
  db.rollback() # nos volvemos al punto de partida

```

## Crear un Cursor

Un cursor es un objeto de Python que le permite trabajar con la base de datos, act√∫a como un identificador para una consulta SQL determinada; permite la recuperaci√≥n de una o m√°s filas del resultado. Para crear un cursor utilizamos el objeto de conexi√≥n de la siguiente forma:

```python

```
{: .nolineno }


### M√©todos de un objeto Cursor

Los siguientes m√©todos del objeto cursor son √∫tiles:

|M√©todo|Descripci√≥n|
|:-----|:----------|
|`execute()`|Ejecuta la consulta SQL como par√°metro de cadena.|
|`executemany()`|Ejecuta la consulta SQL usando un conjunto de par√°metros en la lista de tuplas.|
|`fetchone`|Obtiene la siguiente fila del conjunto de resultado de la consulta.|
|`fetchall()`|Obtiene todas las filas restante del conjunto de resultados de la consulta.|
|`close()`|Cierra el objeto cursor.|


## CRUD de Consola

Ahora vamos hacer un CRUD completo, para que tengamos una aplicaci√≥n de consola. ¬øEntretenido? <i class="fa-regular fa-face-grin-tongue-wink fa-bounce"></i>

Para facilitarnos la existencia, vamos a modularizar el c√≥digo, comenzando por tener en un archivo `db.py` lo relacionado con la creaci√≥n y configuraci√≥n de la base de datos:

```python
import sqlite3
from sqlite3 import Error
import os

CURDIR = os.path.dirname(os.path.abspath(__file__))
FILENAME = "schema.sql"
FILE = os.path.join(CURDIR, "db", FILENAME)

def open_db():
    try:
        con = sqlite3.connect('cars.db')
        return con
    except Error as e:
        print('Error: ', e)

def run_query(sql, params='', multiple=False):

    with open_db() as con:
        cursor = con.cursor()
        try:
            if multiple:
                return cursor.executemany(sql, params)
            else:
                return cursor.execute(sql, params)
        except Error as e:
            print('Error: ', e)

def create_schema():
    with open(FILE, 'r') as sql_file:
        schema_created = run_query(sql_file.read())
        if schema_created.rowcount == -1:
            print("Database created successfully")

if __name__ == "__main__":
    create_schema()
```
{: file="db.py" }


Vamos a resumir que hace el c√≥digo anterior, sobretodo las funciones:

**`open_db`**
: Se encargar√° de crear o abrir la base de datos y se maneja a trav√©s `try/except` para capturar posibles errores.

**`run_query`**
: Esta funci√≥n va a utilizar la conexi√≥n que retorna `open_db` y con ella podemos realizar consultas a la base de datos.

**`create_schema()`**
: Esta funci√≥n va a construir el esquema de la base de datos que tenemos definido en un archivo llamado `schema.sql`.

El archivo que que se encarga de crear la bae de datos se llama `schema.sql` y lo debemos crear, en √©l, se definen las siguientes instrucciones:

```sql

```
